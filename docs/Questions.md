## Вопросы для реализации эксперимента

### 1. О выборе нейросетей и задач

1. **Какую конкретную задачу X выбрать для MVP?** 
   - В документе упоминаются варианты: игра (шахматный эндшпиль, Hex), логические головоломки, текстовая игра "20 вопросов"
   - Для максимальной простоты предлагаю: **простую текстовую игру или классификацию** — например, задачи на стиль рассуждения или ролевые диалоги, где "личность" агента легко измерить
   - **Вопрос:** Подходит ли такой подход, или вы предпочитаете среду с более чёткими action-space (игры)?
Да, такой вариант подходит

2. **Размер моделей для MVP:**
   - Полноценный эксперимент предполагает 1-7B параметров
   - **Для MVP предлагаю:** небольшие модели (до 1B) или даже fine-tuned версии маленьких моделей типа `TinyLlama`, `Phi-2`, `Qwen2-0.5B`
   - **Вопрос:** Есть ли ограничения по вычислительным ресурсам? Какие GPU доступны?
Выбери самую мальнкую возможную модель - доступна RTX 3060Ti

### 2. О разделении серверов S₁ и S₂

3. **Физическое разделение:**
   - Нужно ли реально разделять системы на два физических/виртуальных сервера, или достаточно логического разделения (два отдельных процесса без обмена данными)?
Пока для простоты (у меня нет второго компьютера с GPU) достаточно логического разделения. Но на стадии production необходимо предусмотреть физическое разделение

4. **Общие priors (онтологический граф):**
   - Документ упоминает "общий онтологический граф" как якорь для согласования латентных пространств
   - **Вопрос:** Использовать готовый граф знаний (WordNet, ConceptNet) или создать минимальный граф для задачи?
Используй минимальный граф. Если подойдет готовый - бери его

### 3. О метриках и TDA

5. **Реализация TDA:**
   - Для MVP достаточно ли использовать библиотеку GUDHI/Ripser "из коробки", или нужна кастомная обработка?

Пока минимальный уровень анализа. Главное - скорость

   - В документе предложен LLM-based парсер CoT в AST
   - **Вопрос:** Для MVP использовать LLM для парсинга или начать с простых regex/rule-based эвристик?
Начни с эвристик

### 4. О протоколе эксперимента

7. **Что считать "успехом" MVP?**
   - Вариант A: d_P(s_A, s_B) снижается после работы шамана B_ш (статистически значимо относительно контроля)
   - Вариант B: Качественное наблюдение — "B начинает вести себя похоже на A" по человеческой оценке
   - **Вопрос:** Какой критерий приоритетнее на первом этапе?
Вариант A - полностью объективные статистические метрики успеха переноса

8. **Режим зонда vs базовый перенос:**
   - Документ описывает два сценария: базовый перенос s_A → s_B и зондовую миссию
   - **Вопрос:** С чего начинать MVP — с базового переноса или сразу включить зондовый сценарий?
Начнем с базового переноса. Режим зонда - отдельная задача
